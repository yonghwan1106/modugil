1
전국 통합데이터 7종을 활용하는 과정에서 어려웠던 점은 무엇입니까?
  • API 응답 구조가 7종 데이터마다 상이하여 공통 파싱 로직 설계에 상당한 시간이 소요되었습니다. 특히 응답 필드명과 데이터 타입의 일관성이 부족하여 각 데이터 유형별로 개별 매핑 로직을 구현해야 했습니다.
  • 일부 API의 지자체코드(stdgCd) 체계가 데이터 분류별로 다르게 적용되어, 동일 지역 데이터를 통합 조회할 때 코드 변환 작업이 추가로 필요했습니다.
  • 자전거 API(B551982)는 다른 6종과 베이스 URL 및 인증 방식이 달라 별도의 API 클라이언트를 구현해야 했으며, 이로 인해 통합 데이터 레이어 설계가 복잡해졌습니다.
  • 실시간 데이터의 갱신 주기가 API 문서에 명확히 기재되지 않아, 캐시 TTL 설정 시 시행착오를 겪었습니다. 불필요한 API 호출을 줄이면서도 데이터 신선도를 유지하는 최적값을 찾는 데 어려움이 있었습니다.
  • API 인증키(serviceKey) 발급 후 실제 활성화까지 수 시간의 대기시간이 발생하여, 개발 초기에는 Mock 데이터로 우회하여 작업해야 했습니다.
2
활용하신 데이터에 추가로 필요한 정보가 있었습니까?
  • 교차로 신호등 데이터에 음향신호기 유무 정보가 포함되면, 시각장애인을 위한 보행 안내 기능에 직접 활용할 수 있습니다.
  • 버스 노선 데이터에 저상버스 여부(lowFloor) 필드가 추가되면, 휠체어 이용자 등 교통약자 맞춤 대중교통 안내를 강화할 수 있습니다.
  • 도서관 좌석 데이터에 장애인 전용석/우선석 정보가 포함되면, 교통약자의 공공시설 이용 편의성을 높일 수 있습니다.
  • 각 API 응답 필드에 대한 상세 데이터 사전(코드값 설명, 단위, 허용 범위 등)이 부족하여 필드 해석에 어려움이 있었습니다.
  • 위도/경도 필드명이 lat/lot으로 되어있는데, lot이 longitude를 의미하는지 직관적이지 않아 lng 또는 lon으로의 변경을 권장합니다.

현재 개방된 데이터 외에 신규로 개방되기를 희망하는 공공데이터가 있습니까?
희망 데이터명 필요 이유 / 활용 목적 기대 효과
전국 엘리베이터 · 경사로 위치 정보 교통약자의 건물 및 지하철역 내 수직 이동 경로를 안내하기 위해 엘리베이터, 경사로의 정확한 위치 데이터가 필요합니다. 휠체어 사용자의 이동 경로 최적화 및 접근 불가 구간 사전 안내 가능
장애인 화장실 위치 및 시설 정보 교통약자가 외출 시 주변 편의시설을 사전에 확인할 수 있도록 장애인 화장실의 위치, 설비 정보가 필요합니다. 교통약자의 외출 불안감 해소 및 이동 반경 확대에 기여
전국 무장애 관광지 정보 교통약자의 여행 및 관광 안내 서비스로 확장하기 위해 배리어프리 관광지 데이터가 필요합니다. 교통약자의 사회 참여 및 여가 활동 지원, 관광 접근성 향상
4
공모전 참가 및 데이터 활용에 대한 기타 의견
  • 7종 데이터를 하나의 통합 포털에서 테스트할 수 있는 Sandbox 환경이 제공된다면, API 연동 검증 및 개발 효율성이 크게 향상될 것으로 기대합니다.
  • API 활용 가이드에 실제 요청/응답 예시(JSON 샘플)를 포함하면, 초기 개발 진입장벽이 상당히 낮아질 것입니다.
  • 본 공모전이 교통약자 등 사회적 약자를 위한 데이터 활용에 초점을 맞추고 있어, 공공데이터의 사회적 가치를 실현하는 매우 의미 있는 기회라고 생각합니다.
  • 향후 데이터 품질 개선(결측값 감소, 갱신 주기 단축, 필드 표준화)이 이루어지면, 더욱 실용적이고 안정적인 서비스 개발이 가능할 것으로 판단됩니다.
